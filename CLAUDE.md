## Project Overview

sortpics-go is a photo/video organization tool written in Go. It organizes media files into a chronological directory structure using EXIF metadata with a fallback hierarchy: EXIF → QuickTime → filename → filesystem.

## Architecture

```
File → MetadataExtractor → PathGenerator → DuplicateDetector → ImageRename (atomic copy/move)
```

**Components**:
1. **internal/metadata** - EXIF extraction via ExifTool, datetime fallback, make/model normalization
2. **internal/pathgen** - Generates `YYYY/MM/YYYY-MM-DD/YYYYMMDD-HHMMSS.subsec_Make-Model.ext`
3. **internal/duplicate** - SHA256 hashing, duplicate detection, collision resolution (`_N` suffix)
4. **internal/rename** - Orchestrates pipeline, atomic file operations
5. **cmd/sortpics/cmd** - CLI (Cobra), worker pool, progress tracking

**Key Design Decisions**:
- ExifTool wrapper for 500+ format support (see DECISION.md for rationale)
- Atomic operations with temp files and cleanup on error
- Bounded concurrency with worker pool and context cancellation

## Testing

- Use `github.com/stretchr/testify`: `assert.Equal(t, expected, actual)` and `require.NoError(t, err)`
- Coverage targets: 90%+ overall, 95%+ for core packages
- Test fixtures in `test/testdata/` generated by `make test-fixtures`
- Unit tests mock ExifTool; integration tests use real ExifTool with fixtures

## Code Patterns

- Cross-filesystem moves: Handle `syscall.EXDEV` by falling back to copy+delete
- Atomic operations: `os.CreateTemp()` with UUID names, cleanup on error
- Error wrapping: `fmt.Errorf("context: %w", err)`
- Resource cleanup: `defer file.Close()` and `defer os.Remove(tempFile)`

## Requirements

- Go 1.21+
- ExifTool binary (`exiftool -ver` to verify)
